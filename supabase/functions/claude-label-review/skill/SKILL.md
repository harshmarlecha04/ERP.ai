## HARD CONSTRAINTS (read before every review)

These rules are not optional. If any rule fails, the review is incomplete and the report must not be generated until the rule is satisfied.

1. **ASK BEFORE STARTING.** Before any math pass or report generation, the assistant must ask the user — using `ask_user_input_v0` or plain prose — these two inputs:
   - Weight per gummy (grams)
   - Reviewer name for the report

   Defaults apply ONLY if the user explicitly waives a question or cannot answer (e.g., competitor label without formula access). When a default IS applied, use **3.0 g per gummy** (Pharmvista standard) and reviewer **Pharmvista QA**. Every applied default must be flagged in the report's Assumptions row.

   **Base type and review type are NOT asked.** Base defaults to pectin and is inferred from the "Other Ingredients" line on the label when needed (look for "pectin," "carrageenan," "gelatin," or "agar" — adjust math benchmarks accordingly). Review tone defaults to professional/client-facing.

2. **RUN THE PRE-REPORT CHECKLIST.** Step 3.5's checklist covers every CFR sub-requirement across the five passes. If any checkbox can't be marked done, the review is incomplete. The most-missed items are: botanical plant parts (§101.36(b)(2)(iii)), mineral order (§101.36(b)(2)(i)(A)), elemental mineral declaration, sodium contribution from sodium-form ingredients.

3. **SHOW MATH IN EVERY NUMERICAL FIX.** Every Recommended Fix involving a number must include formula + inputs + result + rounding step. No "recalculate" or "verify with batch sheet" without showing the work.

4. **PLAIN LANGUAGE.** Action-verb-first ("Change X to Y", "Move X above Y"). No insider abbreviations without context.

5. **INTERNAL INCONSISTENCY = ONE FINDING, BOTH SCENARIOS.** When two declared values contradict (Calories vs Total Carb, Total Sugars vs Added Sugars, compound weight vs elemental claim), treat as one finding. Present both calculations and let the user reconcile from the batch sheet.

6. **SERVING SIZE VERIFIED FIRST.** Before any other math, verify: gummies × weight = serving weight, and serving weight ≥ sum of declared components. If the math is physically impossible (e.g., 12 g carbs in a 9.0 g serving — which is 3 × 3.0 g at the Pharmvista default), flag and stop relying on the rest of the panel.

7. **PAGE LENGTH.** Soft target 1 page portrait; up to 2 pages landscape acceptable when math-rich fixes need the room. Never sacrifice math content to hit one page — drop Minors first.

8. **VISUAL TREATMENT.** Apply the color-coded severity badges, count chips, zebra stripes, crimson brand bar, monospace math, and dark header rows per `assets/report-structure.md`. The reference implementation lives at `assets/report-template.js` — copy and adapt that file rather than rewriting docx-js code from scratch. No black-and-white tax-form aesthetic.

This skill produces a focused compliance review of a dietary supplement label against seven specific FDA CFR sections:

1. **21 CFR Part 111** — Current Good Manufacturing Practice (with specific attention to **§111.70** for specifications on quality, purity, strength; and other label-relevant items like lot code and expiration support)
2. **21 CFR 101.36** — Nutrition Labeling of Dietary Supplements (the Supplement Facts panel)
3. **21 CFR 101.36(b) + 101.9(c)/(g)** — Nutrition Math Verification (calorie equation, %DV recalc, rounding, elemental vs salt-weight, base-composition benchmarks)
4. **21 CFR 101.93** — Structure/Function Claims and DSHEA Disclaimer (including disease-claim prohibition under §101.93(g))
5. **21 CFR Part 190** — New Dietary Ingredient (NDI) notifications
6. **21 CFR 101.105** — Principal Display Panel layout and Net Quantity of Contents declaration
7. **21 CFR 101.17** — Ingredient-triggered Warning Statements (iron, psyllium, aspartame, sorbitol, ephedra, etc.)

Plus a separate **Language Review** pass (spelling, grammar, punctuation, symbols) — reported in its own section, **informational only, does not affect the CFR verdict**.

Output: an inline summary in chat plus a single-page `.docx` report saved to `/mnt/user-data/outputs/`.

## What this skill does NOT cover

Out of scope by default — do not include unless explicitly asked:
- Other Part 101 subsections (101.3 Statement of Identity beyond PDP prominence check, 101.4 ingredient list ordering, 101.5 manufacturer info, 101.18 misbranding/professional-use claims, 101.22 colors/flavors, etc.)
- FALCPA / FASTER Act allergen labeling (the "Contains:" statement is FALCPA, not 101.17 — out of scope unless asked)
- FTC substantiation / advertising rules (efficacy dosing claims, etc.)
- Country of origin marking
- Non-US markets (Canada NHP, EU, GCC/SFDA)

If the user wants these added, ask explicitly and run them as a separate pass.

2. Who is the reviewer / what name should appear in the report? (Default "Pharmvista QA" only if the user says so or explicitly skips this question.)

### Step 3.5 — Pre-Report Checklist (MANDATORY before writing the report)

Before moving to Step 4 (Severity classification), explicitly walk through every line below and confirm checked. **If you cannot say "checked" for a line, you have not finished the review.** This is not optional — it is the contract for a complete review.

**§101.36 Supplement Facts panel — format checks:**
- [ ] Title "Supplement Facts" is bold and the largest type in the panel
- [ ] Serving size and servings per container present and math-consistent with PDP count
- [ ] Order of nutrients per (b)(2)(i)(A): Calories → Total Fat → Sat Fat → Trans Fat → Cholesterol → **Sodium** → Total Carbohydrate → Dietary Fiber → Total Sugars → Added Sugars → Protein
- [ ] **Vitamins and minerals listed BEFORE** botanicals/other dietary ingredients (b)(2)(i)(A)
- [ ] **Every mineral declared as elemental amount** with "as [compound]" disclosure (b)(2)(i)–(iii)
- [ ] **Every botanical lists part of plant** (root, leaf, fruit, rind, rhizome, etc.) per (b)(2)(iii) — **CHECK THIS EVERY TIME, it's the most-missed 101.36 finding**
- [ ] Latin/scientific name where required for botanicals
- [ ] Footnote symbol convention: "†" for "Daily Value not established" per (b)(3)(iii)
- [ ] Proprietary blends: total weight gets %DV only if components have DVs; no per-component %DV inside a blend
- [ ] %DV calculation matches declared amount for every line with a DV
- [ ] Heavy bars at title, header, and totals positions

**§101.9(c)/(g) Nutrition Math — load `nutrition-math-verification.md`:**
- [ ] **Serving size verified**: gummies × weight = serving weight; serving weight ≥ sum of declared components (Total Carb + Fiber + Protein + Fat + active solids). Run this FIRST.
- [ ] Calorie equation: (carbs × 4) + (protein × 4) + (fat × 9) matches declared calories within ±5
- [ ] Every nutrient value rounded per 101.9(c) increments
- [ ] Every mineral elemental amount math-checked against declared compound
- [ ] Sodium contribution from all sodium-form ingredients (trisodium citrate, sodium ascorbate, sodium selenite, etc.) accounted for
- [ ] Cane sugar rule: if "organic cane sugar" in Other Ingredients, Total Sugars ≥ 1 g × gummies per serving
- [ ] Added Sugars vs Total Sugars internal consistency in any gummy using cane sugar + glucose/tapioca syrup
- [ ] Base benchmark check: declared values within expected pectin/carrageenan ranges
- [ ] Active load capacity check (sum of declared actives ≤ structural limit for gummy size)

**§101.93 Claims:**
- [ ] DSHEA disclaimer present and asterisk-linked to every S/F claim
- [ ] No explicit disease claim (treats/cures/prevents/mitigates/diagnoses + named disease)
- [ ] No direct drug comparison ("natural Ozempic," "alternative to statins")
- [ ] No disease-class name in product name
- [ ] Hormone-named products (GLP-1, Cortisol, Dopamine) — confirm proper DSHEA + S/F verb framing; NOT a finding when compliant

**Part 111 — cGMP label touchpoints (`cfr-111-label-touchpoints.md`):**
- [ ] Lot code / Exp date placeholder visible (or confirmed added at production)
- [ ] Every standardization claim (e.g., 95% HCA, 98% resveratrol, 4:1 turmeric) flagged for master spec verification per §111.70(d)
- [ ] **Pharmvista gummy base 8-ingredient cross-check** in Other Ingredients
- [ ] Manufacturer or distributor address present
- [ ] "GMP certified" or similar claims substantiated

**Part 190 — NDI screening (`cfr-190-ndi-screening.md`):**
- [ ] Every active screened against known NDI / unfiled NDI list
- [ ] Synthetic-likely ingredients (raspberry ketone, resveratrol, branded peptides) flagged for source documentation
- [ ] Novel branded ingredients verified for filed NDI

**§101.105 — PDP & Net Quantity (`cfr-101-105-pdp-net-quantity.md`):**
- [ ] Statement of Identity present on PDP and prominent
- [ ] "Dietary Supplement" present (PDP or info panel, depending on construction) and prominent
- [ ] Net quantity declaration on PDP (count alone OK for solid count-based items per §101.105(b))
- [ ] Net quantity in correct type size per §101.105(i) (size depends on PDP area)
- [ ] Net quantity in the bottom 30% of the PDP per §101.105(f)
- [ ] Non-mandatory elements (barcode, vignettes, certifications) don't dwarf required elements per §101.15(a)
- [ ] **Don't invent** a "weight required alongside count" violation — count alone is acceptable for solid count items
- [ ] Supplement Facts panel serving size DOES include both count and grams (§101.36(b)(2)(ii)(A)) — this is mandatory, distinct from PDP net quantity

**§101.17 — Warning Statements (`cfr-101-17-warnings.md`):**
- [ ] If iron ≥30 mg elemental Fe/serving: iron pediatric warning present, boxed, bold per §101.17(e)
- [ ] If psyllium present (dry/incompletely hydrated): choking warning per §101.17(d)
- [ ] If aspartame present: PKU warning per §172.804
- [ ] If sorbitol could reach ≥50 g daily: laxative warning per §184.1835
- [ ] If ephedra present: **product is unlawful** under §119.1 — Critical
- [ ] Required warnings properly formatted (bold, boxed, prominent)
- [ ] **Don't invent** missing warnings for ingredients FDA has not mandated (Berberine, Turmeric, Garcinia, etc.) — pregnancy/nursing language for these is industry standard, not §101.17

**Language pass (`language-review.md`):**
- [ ] Spelling, punctuation, ingredient name conventions reviewed

After this checklist, generate the .docx. If any checkbox could not be marked done, address it before proceeding.

### Step 4 — Classify severity

- **Critical** — Unambiguous CFR violation; cannot lawfully ship as-is.
  - Missing Supplement Facts panel.
  - **Explicit** disease claim per 101.93(g): a red-flag verb (treats, cures, prevents, mitigates, diagnoses) combined with a named disease ("treats arthritis," "prevents diabetes," "cures insomnia," "lowers cholesterol").
  - **Direct drug comparison or equivalence claim** ("natural Ozempic," "same as Wegovy," "alternative to statins") — this is what the Veronvy warning letter actually targeted.
  - **Disease-class name in product name** ("DiaBetic Support," "Arthritis Relief").
  - Known unlawful or unfiled NDI under Part 190 (synthetic peptides, FDA-flagged ingredients).
  - **Nutrition math ERROR that misleads the consumer**: calorie equation off by >5 cal, elemental amount impossible from declared compound, "sugar-free" claim with non-zero added sugars, 1990s-era DV used producing a wrong %DV.
  - **Missing Statement of Identity, "Dietary Supplement" descriptor, or Net Quantity declaration** on the PDP per §101.105(a) and §101.3.
  - **Missing FDA-mandated warning** for an ingredient present in the formula: iron warning (§101.17(e)), psyllium choking warning (§101.17(d)), aspartame PKU warning (§172.804), sorbitol laxative warning (§184.1835).
  - **Ephedra** present in formula — banned under §119.1.

- **Major** — Clear CFR-citable issue or borderline language that should be fixed before launch.
  - **Borderline S/F language** ("supports healthy blood pressure" without "already within the normal range").
  - Supplement Facts panel: nutrient order, botanical identification, %DV calculation, missing footnote linkage, missing † for ingredients without DVs.
  - DSHEA disclaimer present but not linked to claim (101.93(d)).
  - Standardization claims not verifiable against master spec under 111.70(d).
  - NDI status uncertain for a medium-risk branded ingredient.
  - **Nutrition math FLAG**: value outside the benchmark range for the declared gummy weight/base, load capacity exceeded, rounding off by one increment, missing "from"/"as" source statement on a mineral.
  - **§101.105 placement / type-size issues**: net quantity not in bottom 30% of PDP, type size below the §101.105(i) table, "Dietary Supplement" present but not prominent.
  - **§101.17 formatting**: required warning present but not boxed, not bold, or in wrong wording.

- **Minor** — Convention-level or low-impact under one of the seven scoped sections.
  - Footnote symbol convention (** vs †).
  - Optional 2,000-calorie footnote on a supplement panel.
  - "Verify against master formula" notes that aren't direct violations.
  - Industry-best-practice warnings missing where no §101.17 mandate exists (only if the user explicitly wants best-practice findings).

**NOT findings (do not flag):**
- **Hormone/biological-target product names** ("GLP-1 Support," "Cortisol Calm," "Dopamine Boost") when paired with proper DSHEA disclaimer, S/F verb, and no drug-comparison or disease language. A naturally occurring hormone is not solely associated with disease and can be referenced in S/F claims. The risk that exists (drug manufacturer C&D letters, class action litigation) is a business risk, not a CFR violation, and is outside this skill's scope.
- **Invented warnings** for ingredients FDA has not mandated under §101.17 — Berberine pregnancy warning, Turmeric warning, Garcinia warning, etc. are industry-voluntary, not CFR-required. Don't fabricate a violation citing §101.17(e) for these.
- **"Count alone is insufficient net quantity"** when count fairly indicates contents — §101.105(b) allows count-only declarations for solid count-based items.
- **"PRO suffix implies professional/prescription use"** — this is §101.18 (misbranding), out of scope unless the user adds it.
- **Any issue that can't be tied to 101.36, 101.9(c)/(g), 101.93, 101.105, 101.17, Part 111, or Part 190.**

**Calibration note:** When in doubt between Critical and Major for a claim issue, default to Major. Critical means "this label is an unapproved drug if printed as-is." That's a high bar; product naming alone (without drug comparisons or disease references) usually does not clear it.

## Key behaviors

- **Cite specifically.** Every finding gets a 21 CFR section (e.g., "21 CFR 101.36(b)(2)"). If a finding can't be cited to 101.36, 101.9(c)/(g), 101.93, 101.105, 101.17, Part 111, or Part 190, drop it — scope is strict.
- **Don't redraft the label.** Findings + one-sentence fix only. The reviewer's job is to identify and cite.
- **Write findings and fixes in plain language.** No insider abbreviations (TSC, HCA, NDI without context), no "recalculate from..." or "verify against..." phrases that don't tell the reader what to actually do. Always lead with the action verb ("Change X to Y", "Move X above Y", "Replace X with Y") and include a specific number whenever possible.

  *Finding — bad:* "Sodium declared 0 mg, but Tri-Sodium Citrate (~26.7% Na) is in Other Ingredients."
  *Finding — good:* "Sodium says 0 mg, but the formula has trisodium citrate, which contains sodium. Expected ~10–15 mg per serving."

  *Fix — bad:* "Recalculate sodium contribution from TSC; declare actual to nearest 5 mg."
  *Fix — good:* "Change Sodium from 0 mg to ~10 mg (confirm exact value from batch sheet)."

- **Show your math.** For any finding that involves a number, the Recommended Fix must include the exact calculation, so the user can verify or redo it from the batch sheet without having to look anything up. Always show: (a) the formula, (b) the inputs, (c) the result, (d) any rounding step.

  *Example — Zinc declaration fix:*
  > Elemental Zn = compound weight × elemental fraction = 7.5 mg × 0.31 = 2.3 mg. %DV = 2.3 ÷ 11 × 100 = 21% → rounds to 20% (nearest 5% under 50%). Restate as "Zinc (as zinc citrate) 2 mg, 20%".

  *Example — Sodium fix:*
  > Trisodium citrate is ~26.7% sodium by weight. Typical use in a pectin gummy: 0.5–1.5% of formula. At 1% TSC and 9.0 g serving (3 × 3.0 g), sodium = 9000 × 0.01 × 0.267 = ~24 mg → round to 25 mg. Restate Sodium to actual value (round to nearest 5 mg).

- **When math is internally inconsistent, do NOT pick a winner.** If two declared values contradict each other (e.g., Calories vs Total Carbohydrate, Total Sugars vs Added Sugars, declared compound weight vs claimed elemental amount), the panel could be wrong in either value or both. Treat it as ONE finding about internal inconsistency, present BOTH calculations as scenarios, and tell the user to reconcile with the batch sheet — do not arbitrarily assert which value is the error.

  *Example — Calorie ↔ Carb inconsistency:*
  > **Finding:** Calories (30) and Total Carbohydrate (12 g) cannot both be correct: 12 × 4 = 48 calories, not 30.
  > **Fix:** Reconcile with the batch sheet. Two scenarios:
  > - If Total Carb 12 g is correct → Calories should be 12 × 4 = 48 → round to 50 (over 50, nearest 10).
  > - If Calories 30 is correct → Total Carb should be 30 ÷ 4 = 7.5 g → round to 8 g.
  > Update both values so they satisfy: Calories = (Total Carb × 4) + (Protein × 4) + (Fat × 9).

- **Be a strict editor.** Findings should be actionable without further interpretation. Bad: "ingredient list has issues." Good: "Sodium row appears after Total Sugars; move it before Total Carbohydrate."
- **Visual + textual.** Always look at the image. Heavy bars, title prominence, and warning type size are only visible in the artwork.
- **Don't over-reach.** If a finding belongs to FALCPA, FTC, or general copyediting, leave it out unless the user asked for those scopes too.
